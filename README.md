# Wraptalk

Google Meet 等の録画 `mp4` / 音声ファイルを、ブラウザ内でポッドキャスト用 mp3 に仕上げるツールです。

主な処理:
- トーク音声にコンプ + ラウドネス正規化
- 先頭にイントロ BGM、末尾にアウトロ BGM を被せる
- トークと BGM が重なる区間は自動で BGM を下げる（ダッキング）
- アウトロのフェードアウトで音量が 0 になった先はカット
- 最終的に mp3 で書き出し

ブラウザ単体で動作し、ファイルはサーバーに送信されません。すべてクライアント側 (`ffmpeg.wasm`) で処理されます。

開発メモは [DEVELOPMENT_NOTES.md](./DEVELOPMENT_NOTES.md)、過去のリファクタ履歴は [REFACTOR_PROGRESS.md](./REFACTOR_PROGRESS.md) を参照してください。

## ブラウザ版（推奨）

`index.html` を HTTP サーバー経由で開くだけです。

- `mp4` / 音声ファイル、`イントロ`、`アウトロ` を選ぶと、波形が表示されます
- 波形上のハンドルをドラッグして、フェード位置・トーク開始位置・終了位置・トーク使用範囲を直感的に調整できます
- `▶ 試聴` ボタンで個別の音源を試聴できます（一時停止位置は保持）
- 各カードの見出し横の色チップが、波形上のハンドル色とリンクしているので調整箇所が一目で分かります
- 配置・設定が終わったら **「設定完了」** ボタンをクリック → モーダルが開いてプレビュー/書き出しのアクションへ
- モーダル内: `オープニングをプレビュー` / `エンディングをプレビュー` で先頭 30s / 末尾 30s のサンプル試聴、`書き出す` で mp3 ダウンロード
- モーダルは × ボタン、背景クリック、`Esc` キーのいずれでも閉じられます

`opening.wav` と `ending.wav` をリポジトリ直下に置いておくと、イントロ / アウトロ未選択時のデフォルト音源として使われます。

### 前提

- `file://` 直開きではなく、HTTP で配信してください
- 配信ルートはリポジトリ直下にして、`/` か `/index.html` を開きます
- Chrome / Edge のデスクトップ推奨です（`ffmpeg.wasm` の安定性のため）
- 60 分超の `mp4` はブラウザのメモリ制約で厳しい場合があります

### ローカルで起動する

バックエンドや専用フレームワークは無く、素の HTML / CSS / JS と `ffmpeg.wasm` だけで動く静的サイトです。リポジトリ直下を HTTP で配信できれば何でも構いません。

```bash
# Python（macOS 標準で入っているので追加インストール不要）
python3 -m http.server 8000

# Node.js
npx serve .
npx http-server . -p 8000

# Ruby
ruby -run -e httpd . -p 8000

# PHP
php -S localhost:8000
```

起動したら http://localhost:8000/ を開いてください。

### 波形上の操作

3 つの波形（トーク / イントロ / アウトロ）には、それぞれドラッグ可能なハンドルが配置されています。

| 場所 | ハンドル | 色 | 役割 |
|------|---------|----|------|
| トーク 波形 上端 | 開始位置 | 🟡 黄 | この時刻より前をカット |
| トーク 波形 上端 | 終了位置 | 🔴 赤 | この時刻より後をカット（`0` のままなら末尾まで使用） |
| イントロ 波形 上端 | トーク開始位置 | 🟡 黄 | イントロ内でトークがかぶり始める時刻 |
| アウトロ 波形 上端 | トーク終了位置 | 🔴 赤 | アウトロ内でトーク終わりが揃う時刻 |
| イントロ / アウトロ 波形 下端 | フェード開始 / 終了 | 🟢 緑 | BGM のフェードアウト区間 |
| 波形本体 | – | – | クリック / ドラッグで再生位置シーク |

各設定カードの見出し横にも、対応する色のチップが付いています（フェードアウト=緑 / トーク開始位置=黄 / トーク終了位置=赤）。

ハンドル上ではカーソルが `ew-resize` に切り替わります。数値入力欄からの直接編集にもリアルタイムで追従します。

### 設定項目

- **目標 LUFS** — 話し声のラウドネス目標値（初期値 `-16`）
- **基本音量** — イントロ / アウトロそれぞれの BGM 音量（初期値 `0.22`）
- **MP3 ビットレート** — 128 / 160 / 192 kbps から選択

### 開発

ピュアな JS 純関数にはユニットテストがあります。

```bash
npm test    # 内部で node --test web/*.test.js (45 ケース)
```

ソースの構成、命名規則、依存関係は [DEVELOPMENT_NOTES.md](./DEVELOPMENT_NOTES.md) を参照してください。

## シェル版 / Mac アプリ版（レガシー）

長尺（60 分超）の mp4 をローカルの ffmpeg で確実に処理したい場合のフォールバックです。`podcast_auto.sh` を直接、または `Wraptalk.app` をダブルクリックで起動します。

### 必要なもの

- `ffmpeg` (`/opt/homebrew/bin/ffmpeg` などにインストール済み)
- `ffprobe`

### 直接実行

```bash
chmod +x ./podcast_auto.sh

./podcast_auto.sh \
  --input /path/to/google-meet-recording.mp4 \
  --intro /path/to/intro.mp3 \
  --outro /path/to/outro.mp3 \
  --output /path/to/episode-final.mp3
```

### よく触るオプション

```bash
./podcast_auto.sh \
  --input /path/to/google-meet-recording.mp4 \
  --intro /path/to/intro.mp3 \
  --outro /path/to/outro.mp3 \
  --output /path/to/episode-final.mp3 \
  --intro-pad 2.5 \
  --outro-overlap 10 \
  --voice-lufs -16 \
  --music-volume 0.18 \
  --mp3-bitrate 128k
```

- `--intro-pad` — イントロ開始から何秒目でトークを始めるか（ブラウザ UI の初期値は `10.0` 秒）
- `--outro-overlap` — アウトロ開始から何秒目をトーク終了地点に合わせるか
- `--voice-lufs` — トークの目標ラウドネス
- `--music-volume` — イントロ / アウトロの BGM 音量

### `Wraptalk.app`（ダブルクリック起動）

ダイアログ操作だけで処理できる Mac アプリ版です。`Wraptalk.app` と `podcast_auto.sh` を同じフォルダに置いて、Mac に `ffmpeg` がインストールされていれば動きます。

- `Google Meet の mp4` を選ぶ
- `intro BGM` を選ぶ（同フォルダに `intro.mp3` があれば使うか聞かれる）
- `outro BGM` を選ぶ（同上で `outro.mp3`）
- 出力先を選ぶ
- 冒頭と末尾の重なり秒数を入れる

## 前提と非機能要件

- トークの細かいカット編集は対象外（トーク使用範囲の前後カットのみ）
- イントロ / アウトロは毎回同じ素材を使う想定
- しゃべり出しは録画ファイルの先頭からそのまま使う前提
